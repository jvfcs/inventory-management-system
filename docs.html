Platform Setup Guide

Follow these initial steps to bootstrap your secure proxy environment.

1 Database Setup

  1. Create a brand new Google Sheet. Name it "IMS Database".
  2. Create 7 tabs and set up the exact Headers in Row 1:
    • Tab 1: Items (Barcode, Make, Model, Category, Min, Spec, Image URL, Status, Description, Price, Image File, Notes, Track Serials)
    • Tab 2: Locations (Location Name, Type, Status, Description, Notes, Image URL, Image File, Space Type, Deployment Location, LBAC Groups)
    • Tab 3: Log (Date, Email, Barcode, From, To, Quantity, Action, Serial Number, Spot, Notes)
    • Tab 4: Users (Email, Last Login, Status, Role, Name, Theme, Locations)
    • Tab 5: Template (Key, Value)
    • Tab 6: Dashboard (Paste into A1: =IFERROR(QUERY({ARRAYFORMULA(TO_TEXT(Log!C1:C)), ARRAYFORMULA(TO_TEXT(Log!E1:E)), Log!F1:G}, "SELECT Col1, Col2, SUM(Col3) WHERE Col1 <> '' GROUP BY Col1, Col2 PIVOT Col4 LABEL Col1 'Barcode'"), "No Transaction Data Found"))
    • Tab 7: Alerts (Paste into A1: =ARRAYFORMULA(QUERY({Items!A2:B, Items!E2:E, IFERROR(VLOOKUP(Items!A2:A, QUERY(Log!A:G, "SELECT C, SUM(F) GROUP BY C"), 2, FALSE), 0)}, "SELECT Col1, Col2, Col3, Col4 WHERE Col4 < Col3 AND Col1 IS NOT NULL LABEL Col1 'Barcode', Col2 'Item', Col3 'Min Threshold', Col4 'Total On Hand'")))
  3. Go to the Template tab and create these rows (Column A = Key, Column B = Value):
    googleClientId | Your Google OAuth Client ID
    driveFolderId | Your Image Storage Drive Folder ID
    licenseKey | The Key provided by your representative
    mothershipUrl | The URL provided by your representative

2 API Proxy Setup

  1. In your Google Sheet, click Extensions > Apps Script.
  2. Open Code.gs, erase the default text, and paste the proxy code provided to you. Click the 💾 Save icon.
  3. Authorize External Requests: Scroll to the bottom of the code and temporarily paste this: 
    function forceAuth() { 
  UrlFetchApp.fetch("https://google.com"); 
  Session.getEffectiveUser().getEmail();
  Session.getActiveUser().getEmail();
}
  4. At the top toolbar, select forceAuth from the dropdown menu and click Run. Complete the security prompts (Review Permissions > Advanced > Go to project > Allow). Delete the function when done.
  5. Click Deploy > New deployment. Select Web App. Execute as: Me. Who has access: Anyone.
  6. Click Deploy and copy your new Web App URL (it must end in /exec).
